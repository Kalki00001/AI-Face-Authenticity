## 🚨 The Problem

With the rise of Generative AI tools like **Stable Diffusion, MidJourney, FaceSwap, and DeepFaceLab**, it is now trivially easy to:

- Generate hyper-realistic **fake faces** indistinguishable from real photos
- Create **deepfake videos** where someone's face is swapped onto another body
- **Fool camera-based security systems** by holding a photo or screen in front of them

> Existing solutions either require expensive GPUs, are cloud-only, or only handle one type of attack. TruthLens PRO solves all three attack vectors locally.

## 🧠 AI Models & Engines

### Engine 1 — MiniFASNetV2 (Liveness Detection)
- **Architecture**: Lightweight CNN designed specifically for Face Anti-Spoofing
- **Source**: Silent-Face-Anti-Spoofing (CVPR 2022 research paper)
- **Optimized**: Converted from PyTorch → **ONNX Runtime** for **3–5× CPU speedup**
- **Speed**: ~40–80ms per frame on CPU
- **Detects**: Photo attacks, video replay attacks, 3D mask attacks
- **Output**: 3-class probabilities — Photo Spoof / Video Spoof / Real Human

### Engine 2 — ViT AI-Image-Detector (Deepfake Detection)
- **Architecture**: Vision Transformer (ViT) fine-tuned on AI vs Real dataset
- **Source**: `umm-maybe/AI-image-detector` (HuggingFace)
- **Runs**: 100% locally after one-time download (~300MB cached)
- **Speed**: ~1–3 seconds per image on CPU
- **Detects**: Images/videos generated by Stable Diffusion, DALL-E, GANs, etc.
- **Output**: Binary — "artificial" or "natural" with confidence score

### Engine 3 — YuNet (Face Detection)
- **Type**: Lightweight OpenCV DNN face detector
- **Speed**: <5ms per frame
- **Output**: Bounding box + 5 facial landmark points for downstream models

### Advanced Signal Processing
- **FFT Moire Analysis**: Fast Fourier Transform detects pixel-grid patterns of digital screens (catches screen replay attacks)
- **3D Head Pose Estimation**: Uses `cv2.solvePnP` with facial landmarks to compute real-time Pitch, Yaw, Roll — proves the object has 3D depth, not a flat photo

## 🎯 Key Features

### Three Analysis Modes
1. **Live Webcam Scanning** — Captures frames every 400ms → AI pipeline classifies in real-time → Animated bounding box HUD overlaid on camera
2. **Photo Upload** — Drag & drop or click to upload → Instant AI-generated verdict → Confidence score + detailed reasons
3. **Video Upload** — Samples 1 frame per second (max 8 frames) → Per-frame analysis → Bar chart breakdown + aggregated verdict

### Dashboard & Telemetry
4. **Threat Level Badges** — Three levels: `VERIFIED` / `SUSPICIOUS` / `THREAT DETECTED`
5. **Animated Confidence Gauge** — Colour-coded bar (green/yellow/red) showing authenticity score
6. **3D Head Pose Display** — Real-time Pitch / Yaw / Roll angles in left panel
7. **FFT Frequency Score** — Moire screen-spoof signal strength
8. **Inference Latency** — Live ms/frame counter showing AI speed
9. **Neural Activity Visualiser** — Animated bars showing inference activity
10. **Recent Scans History** — Last 6 results with timestamps
11. **Export Report** — Downloads full JSON forensic report with all telemetry

## ⚡ Setup & Installation

> ⚠️ This project requires a `.env` file from the owner to run. See note below.

### Backend
```powershell
cd backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt

# One-time: generate the ONNX model
python convert_to_onnx.py

# Start server
python main.py
```

### Frontend
```powershell
cd frontend
npm install
npm run dev
```

Open **http://localhost:5173**

### First Photo/Video Analysis
The ViT model downloads automatically on first use (~300MB, cached at `~/.cache/huggingface/`). After that it runs fully offline.

## 🔐 Security Note

This repository does **not** include the `.env` configuration file. The backend requires a `TRUTHLENS_API_KEY` to start. Without it, the server exits immediately:

```
[ERROR] Missing TRUTHLENS_API_KEY in .env file.
[ERROR] Server cannot start. Contact the project owner.
```
