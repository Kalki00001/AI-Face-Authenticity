## 🚨 The Problem

With the rise of Generative AI tools like **Stable Diffusion, MidJourney, FaceSwap, and DeepFaceLab**, it is now trivially easy to:

- Generate hyper-realistic **fake faces** indistinguishable from real photos
- Create **deepfake videos** where someone's face is swapped onto another body
- **Fool camera-based security systems** by holding a photo or screen in front of them

> Existing solutions either require expensive GPUs, are cloud-only, or only handle one type of attack. TruthLens PRO solves all three attack vectors locally, completely offline, with full user session management.

## 🧠 AI Models & Engines

### Engine 1 — MiniFASNetV2 (Liveness Detection)
- **Architecture**: Lightweight CNN designed specifically for Face Anti-Spoofing
- **Source**: Silent-Face-Anti-Spoofing (CVPR 2022 research paper)
- **Optimized**: Converted from PyTorch → **ONNX Runtime** for **3–5× CPU speedup**
- **Speed**: ~40–80ms per frame on CPU
- **Detects**: Photo attacks, video replay attacks, 3D mask attacks
- **Output**: 3-class probabilities — Photo Spoof / Video Spoof / Real Human

### Engine 2 — ViT AI-Image-Detector (Deepfake Detection)
- **Architecture**: Vision Transformer (ViT) fine-tuned on AI vs Real dataset
- **Source**: `Ateeqq/ai-vs-human-image-detector` (HuggingFace)
- **Runs**: **100% Offline** (`HF_HUB_OFFLINE=1`). Model is cached locally and requires zero internet connection to run.
- **Speed**: ~1–3 seconds per image on CPU
- **Detects**: Images/videos generated by Stable Diffusion, Midjourney, DALL-E, GANs.
- **Output**: Binary classification with aggregated confidence scoring for videos.

### Engine 3 — YuNet (Face Detection)
- **Type**: Lightweight OpenCV DNN face detector
- **Speed**: <5ms per frame
- **Output**: Bounding box + 5 facial landmark points for downstream models

### Advanced Signal Processing
- **FFT Moire Analysis**: Fast Fourier Transform detects pixel-grid patterns of digital screens (catches screen replay attacks)
- **3D Head Pose Estimation**: Uses `cv2.solvePnP` with facial landmarks to compute real-time Pitch, Yaw, Roll — proves the object has 3D depth, not a flat photo

## 🎯 Key Features

### 🔐 Secure Authentication & Database
1. **User Accounts** — Full user registration and login system with secure SHA-256 password hashing.
2. **MongoDB Persistence** — User data and active sessions are stored in a local MongoDB database (`truthlens`).
3. **Session Management** — Token-based persistent sessions keep users logged in.

### 🔬 Three Analysis Modes
4. **Live Webcam Scanning** — Captures frames every 400ms → AI pipeline classifies in real-time → Animated bounding box HUD overlaid on camera.
5. **Photo Upload** — Drag & drop or click to upload → Instant AI-generated verdict → Confidence score + detailed reasons.
6. **Video Upload** — Samples 1 frame per second → Smart thresholding (resists false positives from video compression/text overlays) → Aggregated verdict.

### 💻 Premium Dashboard & Telemetry
7. **Threat Level Badges** — `VERIFIED` / `THREAT DETECTED` / `AI GENERATED`.
8. **Animated Confidence Gauge** — Colour-coded dynamic circular progress.
9. **3D Head Pose Display** — Real-time Pitch / Yaw / Roll angles.
10. **FFT Frequency Score** — Moire screen-spoof signal strength.
11. **Neural Activity Visualiser** — Animated bars showing inference activity.

## ⚡ Setup & Installation

> ⚠️ **Database Requirement:** You must have MongoDB running locally on `localhost:27017` before starting the backend.

### Backend
```powershell
cd backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt

# Start server
python main.py
```

### Frontend
```powershell
cd frontend
npm install
npm run dev
```

Open **http://localhost:5173** and register a new account to access the dashboard!
